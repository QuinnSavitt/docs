---
title: "Primary Operators"
description: "Primary operators are required to be the first token after `ts` , and all statements will evaluate down to one of the following."
---

### `rizz`  [id] [expr]

Assigns the result of `expr` to `id`

```
ts rizz x tun tun tun sahur pmo -> assign value 2 to x <-
```

### `yap`[expr]

Prints the result of `expr` 

### `sybau` [expr]

NoOp. Most useful for in-place operations

### `dih` [expr]

Sets return value of the current function with the value of `expr`.

### Additional Primary Operators

More complex or specific primary operators can be found in [Additional Operators](/additionalOperators) and [List Operators](/listOperators).
